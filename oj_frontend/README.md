# 教学型 OJ 系统 · 前端

React 18 + Vite + Ant Design 5 + Monaco Editor 实现，对接 `oj_backend` 的 REST API。

**设计方向**：纸面学术风——暖纸底色、松绿主色、思源宋体标题、JetBrains Mono 代码字体，
像一份印刷讲义，同时保持做题工具所需的信息密度与清晰度。

---

## 运行

前置：后端已按其 README 跑起来（Django 在 `127.0.0.1:8000`，Celery worker 在运行）。

```bash
npm install
npm run dev        # http://localhost:5173
```

Vite 已配置代理：`/api` 与 `/media` 转发到 `127.0.0.1:8000`，开发环境无需任何跨域配置。
（Monaco 编辑器与字体走 CDN，需要本机能联网。）

生产构建：`npm run build`，产物在 `dist/`，由 Nginx 托管并将 `/api` 反代到后端即可。

## 快速体验路径

1. 后端先执行 `python manage.py seed_demo`（生成示例题 #1 A+B）；
2. 前端注册账号 → 后端 runserver 控制台里复制验证链接中的 token → 打开
   `http://localhost:5173/verify-email?token=粘贴token` 完成验证 → 登录；
3. 进入题库 → 「1. A + B Problem」→ 写代码提交，观察判题结果实时刷新；
4. 教师账号可从右上角头像菜单进入「出题管理」新建题目并上传测试数据 ZIP。

## 功能总览（已实现）

### 认证
- 登录（JWT，自动续期）、注册、邮箱验证页、未登录态引导
- **找回密码**：申请重置邮件 → 邮件链接 `/reset-password?token=…` 设置新密码

### 题库与做题（核心）
- **题库列表**：搜索、难度筛选、做题状态标记（已解/尝试过）、通过率、标签、分页
- **做题页**：
  - 左侧三选项卡：题目描述（Markdown + LaTeX + 样例对照）/ 题解 / 提交记录
  - 右侧 Monaco 编辑器：三语言切换、默认模板、草稿自动存本地（按题×语言）、重置
  - 提交 → 轮询判题 → 结果面板：状态、用时/内存/得分、**首个未通过测试点提示**、
    编译错误回显、逐测试点表格（样例点可展开看 输入/期望/实际 三栏）
  - 题解：列表与详情（审核状态、驳回理由）、写题解（提示需先 AC）
  - 提交记录：我的提交 + 详情抽屉（结果 + 源代码）

### 竞赛
- **列表**：状态/赛制/时长/人数、报名（公开赛与密码赛）、教师可创建
- **详情页**：信息卡 + **实时倒计时**、说明 / 赛题 / 排行榜 / 管理 选项卡
- **赛内做题**：复用做题页并加赛事顶栏（倒计时），提交计入该场竞赛
- **实时排行榜**：ACM 榜（罚时、首杀气泡）与 OI 榜（按分），**封榜**后显示封榜态、
  本人行高亮、前三名奖牌；进行中每 20s 自动刷新
- **创建/编辑竞赛**：时间区间、赛制、班级赛 / 密码赛、封榜时长
- **竞赛管理**（教师）：添加赛题（标号/分值）、私有赛名单维护

### 讨论
- **列表**：发帖（Markdown）、按标题搜索、回复数与最后活跃时间
- **详情页**：正文 + **楼层回复 + 楼中楼**、回复指定楼层、删除自己的回复/帖子、
  管理员下架 / 恢复

### 班级（教学）
- **列表**：卡片网格，教师建班、学生凭邀请码加入
- **详情页**：概览（邀请码复制）/ 成员（教师可移除）/ 作业（教师布置、自动通知全班）

### 出题管理（教师）
- **题目列表**：含隐藏草稿，编辑 / 测试数据 / 删除入口
- **题目编辑**：全字段表单（题号、难度、可见性、允许语言、时空限制、比对模式与浮点精度、
  标签、题面三段、样例、提示、来源、Special Judge 代码）
- **测试数据**：ZIP 拖拽上传（清空重建 / 追加）、测试点分值与「样例可见」批量保存、
  单点删除 / 清空

### 个人
- **个人主页**：资料卡 + 统计（通过数/提交数/通过率）+ **年度提交热力图** + 近期提交
- **个人设置**：头像 / 简介 / 默认语言、**赛后代码公开开关**、发送密码重置邮件

### 通知
- 导航栏铃铛、未读数（每分钟刷新）、近期通知、点击跳转并标记已读、全部已读

## 已知设计权衡

- 个人主页展示当前登录用户自己（后端未提供按用户名的公开主页接口）；
- 赛内做题复用 `/problems/:id` 拉题面，**赛题需将可见性设为「公开」学生方可打开**，
  提交携带 `contest` 计入排行榜；
- 出题列表不显示可见性（列表序列化器未返回该字段），可见性在编辑页设置与查看；
- 编辑题目时不回显已有 SPJ 源码（详情接口出于安全不返回），**留空保存即保留原代码**；
- 竞赛 / 班级的管理入口对教师显示，非创建者由后端 403 兜底；
- 实时排行榜采用轮询（进行中 20s），未上 WebSocket——教学场景下足够且部署简单。

## 目录结构

```
src/
  api/               # axios 客户端（JWT 自动刷新）+ 全部接口封装（58 个）
  auth/              # 登录态 Context
  components/        # 布局导航、通知铃铛、标签、Markdown、提交热力图
  pages/             # 登录/注册/验证/找回密码、题库、竞赛/讨论/班级列表、个人主页/设置
  pages/solve/       # 做题页：题面/编辑器/结果/题解/提交记录
  pages/contest/     # 竞赛详情/编辑、排行榜、竞赛管理
  pages/discussion/  # 讨论详情（楼层回复）
  pages/classes/     # 班级列表/详情
  pages/manage/      # 教师出题：题目列表/编辑、测试数据面板
  utils/             # 判题状态映射、语言与模板、竞赛时间/状态工具
```
