# 更新日志（Changelog）

本文件记录项目所有值得留存的变更。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

变更类型：**新增**（Added）、**变更**（Changed）、**修复**（Fixed）、**移除**（Removed）、**安全**（Security）。

> 说明：本项目早期以“批次”方式迭代，下述版本号与日期为整理归档时按时间线追认，用于后续规范化维护。

---

## [0.10.1] - 2026-07-20

### 修复
- **OI 评测计分**：修复非捆绑题“遇到第一个错误测试点就停止评测”导致部分分偏低的问题。现在 OI 竞赛与练习会跑完所有测试点，按“通过的点得分、不通过的不得分”正确计分；仅 ACM 竞赛保留“首错即停”（排名只看是否通过）。

### 变更
- **题号改为从 1 开始、无前导零**：新建题目题号为 1、2、3 …（不再补零为 00001）；已有题号经迁移去零（`00001` → `1`）。题库按题号顺序排列。

## [0.10.0] - 2026-07-20

### 新增
- **管理员测评记录**：新增「测评记录」页（`/admin/submissions`），管理员可查看全站所有提交（含题库练习、班级作业、竞赛），支持按用户名 / 题号 / 结果筛选，点击查看逐测试点结果与源代码。
- **隐藏测试点对管理员可见**：管理员查看提交详情时，可看到每个测试点（含隐藏点）的输入、期望输出与判定结果；普通用户仍仅能看到样例点。
- **竞赛排行榜下钻**：管理员可点击排行榜中的选手姓名（看其本场全部提交）或某题的分数格（看该题的提交），跳转到测评记录对应筛选。
- 提交列表新增「来源」列：题库 / 竞赛名 / 作业名。

### 变更
- **Django 自带后台迁移到 `/django-admin/`**：避免与前端 `/admin/*` 路由冲突，修复刷新页面时错误跳到 Django 登录页的问题。
- **收紧提交可见性**：普通用户只能查看自己的提交；竞赛创建者可查看自己竞赛内的提交；管理员可查看全部。
- **降低存储占用**：测试点的输入 / 期望输出不再随每次提交冗余存储，改为查看详情时按需从题目测试文件读取（这两项对每道题固定，无需逐份存）；每个测试点仅保存“选手输出”，供管理员审阅隐藏点，普通用户仅见样例点。

## [0.9.1] - 2026-07-19

### 移除
- 移除根目录下的一键脚本（`update.bat`，以及 `启动 / 停止 / 部署 / 局域网部署` 的 `.bat` 与 `.sh`）：其作用仅为封装 Docker 命令，改为直接使用 `docker compose up -d --build` / `docker compose down`（详见 `README.md`）。
- 移除 `README-部署.md`：部署与运维说明已并入 `README.md` 与 `开发文档.md`。

### 新增
- 新增 `README.md`（项目说明与快速开始）、`CHANGELOG.md`（本文件）、`.gitignore`（排除 `.env`、媒体数据与构建产物，避免密钥 / 数据误入库）。

### 变更
- 文档规范化：将原「更新说明」与「修复说明」合并为本 `CHANGELOG.md`；精简 `开发文档.md` 的文档索引与根目录清单。

## [0.9.0] - 2026-07-19

### 变更
- **题号改为系统自动分配**：新建题目时按现有最大数字题号 +1，左补零为 5 位（`00001`、`00002` …）；出题页题号栏改为只读，接口层同样禁止设置或修改，避免人为改乱编号。示例题「A + B Problem」规范为 `00001`。

### 修复
- 新增数据迁移，将历史数字题号统一补零为 5 位，保持全站编号一致（原有链接 `/problems/1` 变为 `/problems/00001`，站内跳转自动适配）。

## [0.8.1] - 2026-07-17

### 修复
- **修复因文件被误覆盖导致后端崩溃、整站无法登录的问题**：`apps/problems/views.py` 曾被误写入 `contests/views.py` 的内容，触发 `ImportError` 使 gunicorn 无法启动、所有请求（含登录）无响应；现已恢复为正确的题目视图。
- 补全「赛中隐藏成绩与榜单」开关此前仅落一半的实现（模型字段 / 序列化器 / 榜单接口 / 提交封存 / 出题表单开关）。

### 变更
- 恢复前端离线化：Monaco 编辑器从本站 `/vs` 加载、字体使用系统内置，彻底不依赖公网 CDN，适配内网 / 无外网环境。

## [0.8.0] - 2026-07-14

### 新增
- **竞赛「赛中隐藏成绩与榜单」开关**（按场设置，默认关闭）：开启后竞赛进行中，普通选手提交只显示「已提交」，看不到评测结果与成绩，排行榜仅创建者 / 管理员可见，竞赛结束后自动公开。适合考试 / 正式赛。
- **赛内做题页**新增赛题切换器（上一题 / 下一题 / 下拉直选）与内嵌排行榜抽屉（进行中自动刷新、遵守封榜）。

### 变更
- **判题沙箱镜像下载改为多源自动回退**（go-judge 二进制），加入连接超时与重试，解决单一 GitHub 代理失效导致的构建卡死；另附离线构建方案 `deploy/gojudge.offline.Dockerfile`。

### 修复
- 修复「仅竞赛可见」题目在竞赛内打不开（题目可见性放行选手可访问竞赛中的赛题）。
- 修复局域网 `http` 访问下「复制」按钮无效（样例复制、班级邀请码复制改用兼容 `http` 的实现）。

## [0.7.2] - 2026-07-08

### 修复
- 修复内网 / 多机按 IP 访问时**又慢又登不上、做题页一直转圈**：根因为前端从公网 CDN 加载 Google Fonts 与 Monaco，且 `ALLOWED_HOSTS` 未包含服务器 IP 致 `/api` 返回 400。已移除公网字体外链、Monaco 本地化、放行相应主机。

### 变更
- 升级流程移除危险的 `docker compose down -v`（会连同数据库与题目数据一并删除），统一改用 `docker compose up -d --build`，升级时保留数据。

### 移除
- 清理 `oj_backend/` 下与根目录重复的开发用 compose 文件，以及冗余的测试数据副本。

## [0.7.1] - 2026-07-05

### 修复
- 修复进入「测试数据」界面 / 上传测试数据的入口异常（算法题标签稳定显示；填空题在编辑页内联显示上传区）。
- 修复测试数据列表因组件漏导入导致的整页白屏。

## [0.7.0] - 2026-07-02

- 题面「运行」默认运行样例 1；样例区新增**一键复制**输入 / 输出。

### 移除

## [0.6.0] - 2026-06-27

### 新增
- **捆绑测试 / 子任务组**：测试点可设子任务组，组内全部通过才得该组分数，任一失败则该组 0 分；提交结果页显示子任务汇总，测试点明细新增「子任务」列。

## [0.5.0] - 2026-06-23

### 新增
- 题面**在线运行**（自定义输入自测，编译并运行一次直接返回结果，不计入提交记录）。
- 教师**批量将学生加入班级**（粘贴用户名，返回成功 / 已在班 / 未找到明细）。
- **班级作业排行榜**（按解题数、末次 AC 时间排序，按真实姓名展示），教师可查看学生在作业题上的全部提交与源代码。

## [0.4.0] - 2026-06-18

### 变更
- **账号体系改造**：关闭自助注册与找回 / 改密（前后端均移除相关页面与路由，直接调用 API 亦无法自助注册或改密），账号由管理员统一创建 / 批量建号 / 重置密码。

### 新增
- 用户名支持中文；新增「真实姓名」字段，竞赛排行榜、班级成员、个人主页按真实姓名展示（缺失时回退用户名）。

## [0.3.0] - 2026-06-13
